# サービスの全体像

VER-Lib は、**Web の管理画面**・**Discord**・**VRChat** の 3 つが連携して動きます。それぞれの役割を押さえておくと、以降の各機能が理解しやすくなります。

## 3 つの入り口

- **Web の管理画面** — イベント運営・スタッフ・参加者それぞれに専用の画面があります。運営はイベント設定や抽選・割り振りを、スタッフは予定管理を、参加者はイベント探しや応募をここで行います。
- **Discord** — 告知や当落などの通知が届く場所であり、参加者はイベント案内担当（Raspis）に相談したり、スラッシュコマンドでイベントを検索したりできます。多くのイベントコミュニティが日常的に使う場所に、そのまま溶け込みます。
- **VRChat** — 実際にイベントが開催される場所です。参加者の割り振り先となるワールドのインスタンスや、Invite リクエストによる参加申請がここに関わります。

## イベント 1 回の流れ

1 つのイベントは、おおむね次の流れで進みます。VER-Lib はこの各ステップを支えます。

```text
告知・募集  →  応募（いいね／応募）  →  締切・集計  →  抽選
                                                        │
      当日運営  ←  当落通知  ←  インスタンス割り振り（組分け）
```

1. **告知・募集** — 運営がイベントを告知し、参加者の募集を始めます。
2. **応募** — 参加者は気になるイベントに「いいね」を付け、開催回ごとに応募します。
3. **締切・集計** — 締切とともに応募者を集計します。VRChat の Invite リクエストや外部 CSV からの取り込みにも対応します。
4. **抽選** — 集めた応募者から、公平な抽選で当選・補欠・落選を決定します。
5. **インスタンス割り振り（組分け）** — 当選者やスタッフを、定員・ロールを守りながら各インスタンスへ割り振ります。
6. **当落通知** — 抽選結果を参加者へ Discord で通知します。
7. **当日運営** — 割り振りをもとにイベントを開催します。

## ワークフローがすべてをつなぐ

上の流れは、運営が**ワークフロー**として一度組み立てておけば、締切・抽選・通知などを自動で進められます。「毎回手作業でやっていたこと」を仕組みに置き換えるのが VER-Lib の中心的な考え方です。

各機能の詳しい内容は [できること](03_what_can_do.md) を、繰り返し出てくる言葉の意味は [基本用語](04_glossary.md) を参照してください。
