# ワークフロー（自動化の中核）

**ワークフロー** は、イベント進行を「トリガー」と「アクション」の組み合わせで組み立て、自動で実行する仕組みです。運営作業の多くは、このワークフローの中に配置します。

## 考え方

「フォーム投稿 → 応募締切 → 抽選 → 組分け → スレッド作成 → 通知」といった流れを、ステップを並べて表現します。設定した条件が満たされると、各アクションが順に実行されます。

## 代表的なアクション

- **参加集計** — スタッフ・キャストの参加を、ロールごとに取り込みます。
- **応募集計** — ゲストの応募を集計します（VRChat の Invite リクエスト / CSV 取り込み）。
- **抽選** — 応募者から当選・補欠・落選を決めます。
- **インスタンス割り振り** — 参加者を各インスタンスへ配置します。
- **スレッド作成** — Discord スレッドを自動作成します。
- **メッセージ送信・通知** — テンプレートを使って告知・通知します。

各アクションの詳しい内容は、以降のページで扱います。繰り返し実行（定期タスク）の設定については [メッセージ・通知](07_messaging.md) を参照してください。
