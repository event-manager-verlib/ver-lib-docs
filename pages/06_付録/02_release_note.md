# リリースノート

VER-Lib の新機能や改善のお知らせです。ここに掲載する内容は、各アプリ（イベント管理 / スタッフ / ゲスト）を開いたときにも、お知らせとして表示されます。各項目の末尾は対象アプリを表します。


## 0.5.0（2026-07-03）マイページ刷新 + スキ済み表示の修正

- **新機能**（全アプリ）— ポータルのトップページをマイページ化しました（アカウント情報・表示名の編集・各アプリへの導線・退会）。
- **新機能**（全アプリ）— 各アプリのアカウントメニューに「マイページ」への戻りボタンを追加しました。
- **修正**（イベント管理）— ログイン中の一覧・検索でスキ済みのイベントが表示されない不具合を修正しました。

## 0.4.0（2026-07-02）スキイベントカレンダー追加 + Discord 操作の安定性修正

- **新機能**（ゲスト）— スキしたイベントをカレンダーで確認できる「カレンダー」ビューを追加 (ヘッダーから切替可能)。
- **新機能**（スタッフ）— 週表示・スケジュールパネルにスキしたイベントを重ねて表示できるようになりました。
- **新機能**（全アプリ）— アカウントメニューに「支援する」リンクを追加しました。
- **新機能**（ゲスト）— ログイン時、イベント検索の並び順をあなたの参加・スキ履歴に合わせてパーソナライズするようになりました。
- **修正**（全アプリ）— Discord のボタン操作がタイムアウトして完了しない問題を修正しました。
- **修正**（全アプリ）— 同じ DM 通知が二重に送信されることがある問題を修正しました。
- **修正**（イベント管理）— 抽選・割当処理がまれに重複実行される問題を修正しました。
- **修正**（イベント管理）— 日付入力で無効な日付が意図せず別の日付に変換される問題を修正しました。

## 0.3.0（2026-06-30）メッセージ通知・アカウント復帰フロー追加 + スタッフ機能の重要修正

- **新機能**（イベント管理）— メッセージ通知「今日のTopics」を追加。締切前の管理対象メッセージを、本文プレビュー + 締切日時 + 元メッセージへのリンク付きで定期的にお知らせします。
- **新機能**（全アプリ）— 退会後のアカウント復帰フローを追加。退会済みアカウントで再ログインすると、データを復旧して復帰するか新規作成するかを選べます (これまで退会から30日間はログインできませんでした)。
- **新機能**（スタッフ）— スタッフ操作ページに magic-link (専用リンク) でアクセスできるようになりました。
- **新機能**（全アプリ）— アカウントメニューに「マニュアル」ボタンを追加し、操作マニュアルへすぐアクセスできるようにしました。
- **修正**（スタッフ）— スタッフ予約がセッション作成時に登録されないことがある重大な不具合を修正しました (予約データの消失)。
- **修正**（スタッフ）— スタッフ割り当て画面でスタッフ名が表示されない問題を修正し、実名表示を復旧しました。
- **修正**（スタッフ）— スタッフのスケジュール表示・カレンダー同期 (.ics) が空になることがある問題を修正しました。
- **修正**（イベント管理）— Discord メンバーのロール変更時にエラーになる問題を修正しました。
- **修正**（イベント管理）— Discord 通知が長文で送信に失敗する問題、および embed 通知が本文と二重に表示される問題を修正しました。
- **修正**（全アプリ）— Discord 連携の同意 (consent) DM が届かない・同意後に元の操作へ自動復帰しない問題を修正しました。
- **修正**（イベント管理）— 退会済みアカウントをグループに招待した際に案内 DM が届かない問題を修正しました。
- **修正**（イベント管理）— テンプレート編集で変数の入力候補がクリックで挿入できない・モーダル内でスクロールできない問題を修正しました。
- **修正**（全アプリ）— エラー画面・ヘッダーの表示を共通化し、UI の一貫性を改善しました。

## 0.2.2（2026-05-24）Like 表示修正 + 内部基盤微調整

- **修正**（ゲスト）— Like (いいね) 一覧で自分が押したイベントが表示されない問題を解消 (= 内部 ID 形式の不整合を是正)。
- **修正**（イベント管理）— イベント管理画面の reverse-lookup API が、特定の Discord アカウント切替時にエラーになることがあった問題を解消 (Bug 44)。

## 0.2.0（2026-05-16）認証基盤刷新 + 大量バグ修正

- **新機能**（全アプリ）— ユーザー識別子を UUID 軸に統一する認証基盤の刷新を完了 (Phase C-5 → C-7)。内部 ID と公開 ID を分離し、将来の機能拡張時にプライバシーと拡張性を両立できる構造に。
- **修正**（全アプリ）— ログアウト後に最大 16 分間ログインできなかった問題を解消。
- **修正**（ゲスト）— ゲストユーザーがイベントページを開けない場合がある不具合を修正。
- **修正**（イベント管理）— OGP カードがグループ ID のみ表示される / 生成失敗していた複数のイベントグループで、Discord 共有時のカード表示を復活。
- **修正**（イベント管理）— システムテンプレート編集時の保存が無反応になっていた問題を解消、編集内容が新規のカスタムテンプレートとして正しく保存されるように。
- **修正**（スタッフ）— ワークフロー進行中セッションの「参加」「辞退」操作が無反応になることがあった問題を解消。
- **修正**（イベント管理・スタッフ）— ラインナップ編集の「開催回数」変更が反映されていなかった問題を修正。
- **修正**（イベント管理）— admin 権限付与済のイベントグループが Web UI 上で消失していた問題を解消。
- **修正**（イベント管理）— イベント招待後のメンバー表示名が ID のままだった問題を修正、VER-Lib アカウント名で正しく表示。
- **修正**（イベント管理）— 「イベント実行」アクションが本来の開始時刻より早く発火していた問題を解消、設定通りの時刻に正確に実行。
- **修正**（全アプリ）— Discord OAuth レートリミット時にログインが長時間応答しなくなる事象を解消、適切なエラー応答 + 自動リトライ。
- **新機能**（全アプリ）— 監査ログ基盤を拡充。各サービスの操作履歴を統一的に audit-DB に永続化、今後のインシデント対応の基盤を強化。
- **新機能**（イベント管理）— 古い不要データの自動削除機構を起動時自動構築化、データ蓄積による性能劣化を予防。
